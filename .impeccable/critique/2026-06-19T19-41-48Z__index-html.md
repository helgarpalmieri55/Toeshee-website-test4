## Anti-Patterns Verdict
**LLM:** Not slop — authored, opinionated, sourced proof, disciplined palette, hand-tuned motion. Residue: the "Four Patterns That Separate X From Y" insight-headline formula reads templated; round-number proof ("Live across 5 sectors", "100+/3") invites methodology questions.
**Detector:** exit 0, clean.
**Overlays:** not available (no browser automation) — live contrast over video and mobile remain source-only judgments.

## What's Working
1. The accessibility pass is real craft, not cosmetic: Escape + focus return, global :focus-visible, inline aria-describedby errors, honest async, reduced-motion that reveals via showAll().
2. Proof discipline — every number carries a source; the most persuasive thing on the page for a fintech evaluator.
3. Brand coherence under constraint — one accent, two fonts, oklch ramp; the 404 reads as the same product without sharing a stylesheet.

## Priority Issues (remaining)
**[P1] Accordion ARIA incomplete.** `.sol-trigger` buttons have aria-expanded but no aria-controls; the `.sol-bi` panels have no id/role=region/aria-labelledby. SR users hear "expanded" but not what opened. This is what holds Recognition at 3. *Fix:* id + aria-controls + role=region/aria-labelledby.
**[P1] Breadcrumb contrast (REGRESSION from the a11y pass).** Breadcrumb links use --hi3 (oklch 50%) on the dark phero ≈ 3.5–4:1 — under AA for 0.75rem text. *Fix:* lift breadcrumb links to --hi2, reserve --hi3 for separators.
**[P2] Nav anchor vs page-link ambiguity.** Home nav "Case Studies/Insights/Careers" are #anchors but real pages elsewhere — same label, two behaviors. Holds Recognition + cognitive-load. *Fix:* link home-nav items to canonical pages, or mark in-page anchors.
**[P2] Dropdown aria-expanded desync (REGRESSION).** :focus-within now reveals the panel, but aria-expanded only flips on click/Escape/outside-click — tabbing out leaves it stale. *Fix:* focusout handler on .drop sets aria-expanded=false when focus leaves.
**[P3] role="menu"/"menuitem" is the wrong pattern** for nav links (promises arrow-key menu semantics the JS doesn't implement). *Fix:* remove the roles; plain links in the labelled nav.
**[P3] Possible form double-announcement** — role=alert summary + per-field described-by + focus jump is verbose. *Fix (optional):* make the alert a count ("3 fields need attention") or drop it.
